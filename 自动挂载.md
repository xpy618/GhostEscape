# GhostEscape 自动挂载系统全解析

## 一、类的继承层级

```
Object                          type_ = NONE（默认）
├── ObjectScreen                type_ = OBJECT_SCREEN（init 设置）
│   └── ObjectWorld             type_ = OBJECT_WORLD（init 设置）
│       ├── Actor               (不改变 type_)
│       │   ├── Player          → OBJECT_WORLD
│       │   └── Enemy           → ENEMY（Enemy::init 覆盖为 ENEMY）
│       ├── Effect              → OBJECT_WORLD
│       ├── Spell               → OBJECT_WORLD
│       └── BgStar              → OBJECT_WORLD
├── ObjectAffiliate             type_ = NONE
│   ├── Sprite / SpriteAnim
│   ├── Collider
│   └── AffiliateBar
├── MoveControl                 type_ = NONE
├── Stats                       type_ = NONE
├── Timer                       type_ = NONE
└── Spawner                     type_ = OBJECT_WORLD
```

## 二、Scene 的三容器路由

`Scene::addChild()` 根据 `ObjectType` 把对象分入三个容器：

```cpp
// scene.cpp:87-101
void Scene::addChild(Object *child) {
    switch (child->getType()) {
        case OBJECT_WORLD:
        case ENEMY:        → children_world_   (vector<ObjectWorld*>)
        case OBJECT_SCREEN: → children_screen_  (vector<ObjectScreen*>)
        default:            → children_         (基类 vector<Object*>)
    }
}
```

### 以 SceneMain 为例，三个容器实际装了这些对象：

```
┌─────────────────────────────────────────────────────────────┐
│ SceneMain (继承 Scene)                                       │
│                                                             │
│ children_world_  (世界坐标，渲染时减去相机偏移)                │
│   ├── BgStar          OBJECT_WORLD  背景星空                 │
│   ├── Player          OBJECT_WORLD  玩家                     │
│   ├── Spawner         OBJECT_WORLD  刷怪器                   │
│   ├── Effect (多个)   OBJECT_WORLD  生成/死亡特效             │
│   └── Enemy (多个)    ENEMY         敌人                     │
│                                                             │
│ children_screen_ (屏幕坐标，无视相机，UI层)                   │
│   ├── HUDStats        OBJECT_SCREEN  血条/蓝条               │
│   ├── HUDText         OBJECT_SCREEN  分数文字                │
│   ├── HUDButton ×3    OBJECT_SCREEN  暂停/重开/返回按钮       │
│   └── UIMouse         OBJECT_SCREEN  自定义鼠标光标           │
│                                                             │
│ children_ (基类容器，无坐标概念)                               │
│   └── Timer (end_timer_)  NONE                               │
└─────────────────────────────────────────────────────────────┘
```

**关键点**：`ObjectAffiliate`、`MoveControl`、`Stats` 等 NONE 类型对象**不直接挂在 Scene 上**，它们挂在各 Actor 自己的 `children_` 里。

## 三、Player 的"子树"

Player 自己挂在 Scene 的 `children_world_` 里，但它内部还有一棵子树：

```
Player (OBJECT_WORLD, 在 Scene::children_world_)
  └── children_ (基类容器, NONE 对象):
        ├── SpriteAnim (idle)     精灵动画
        ├── SpriteAnim (move)     移动动画
        ├── Collider              碰撞体
        ├── Stats                 HP/MP 数据
        ├── WeaponThunder         武器系统
        ├── Timer (flash)         受击闪烁计时
        └── MoveControl           移动控制 ← 就是它
```

Enemy 同理：

```
Enemy (ENEMY, 在 Scene::children_world_)
  └── children_:
        ├── SpriteAnim ×3 (normal/hurt/die)
        ├── Collider
        ├── Stats
        ├── AffiliateBar         头顶血条
        └── MoveControl (可选，Spawner 动态添加)
```

## 四、主循环 → 递归遍历的完整执行顺序

以 `SceneMain` 一帧为例：

### 1. handleEvents 阶段

```
Game::run()
  └→ SceneMain::handleEvents(event)
       ├→ Scene::handleEvents(event)
       │    │
       │    ├─ 1️⃣ children_screen_ 遍历（UI 优先响应）
       │    │    HUDStats → HUDText → HUDButtons → UIMouse
       │    │
       │    ├─ 2️⃣ children_ 遍历（基类容器）
       │    │    Timer (end_timer_)
       │    │
       │    └─ 3️⃣ children_world_ 遍历
       │         ├─ BgStar::handleEvents()
       │         ├─ Player::handleEvents()
       │         │    └→ Actor::handleEvents()
       │         │         └→ Object::handleEvents()
       │         │              └→ 遍历 Player.children_:
       │         │                   ├─ SpriteAnim::handleEvents()
       │         │                   ├─ WeaponThunder::handleEvents()
       │         │                   └─ MoveControl::handleEvents()
       │         │                        ← 这里处理 WASD 键盘事件！
       │         ├─ Spawner::handleEvents()
       │         ├─ Effect::handleEvents()
       │         └─ Enemy(们)::handleEvents()
       │
       └→ 返回给 Game
```

注意 `Scene::handleEvents` 的顺序是 **screen → children_ → world**。UI 层优先处理，如果 UI 消费了事件（return true），后续不再传递。暂停时 world 层不处理事件。

### 2. update 阶段

```
Game::run()
  └→ SceneMain::update(dt)
       ├─ checkTimeManage(dt)  // 快慢放
       │
       ├─ Scene::update(dt)
       │    │
       │    ├─ 1️⃣ Object::update(dt) → children_ 遍历
       │    │    └→ Timer (end_timer_)
       │    │         └→ Object::update()
       │    │              ├ 处理 object_to_add_ → 移入 children_
       │    │              └ 遍历 children_ → child->update(dt)
       │    │
       │    ├─ 2️⃣ children_world_ 遍历（暂停时跳过）
       │    │    │
       │    │    ├─ BgStar::update(dt)
       │    │    │    └→ ObjectWorld::update()
       │    │    │         └→ render_position_ = worldToScreen(position_)
       │    │    │              └→ ObjectScreen::update()
       │    │    │                   └→ Object::update()
       │    │    │                        ├ 处理 BgStar 的 object_to_add_
       │    │    │                        └ 遍历 BgStar.children_ → update
       │    │    │
       │    │    ├─ Player::update(dt)
       │    │    │    └→ Actor::update(dt)
       │    │    │         └→ ObjectWorld::update(dt)
       │    │    │              └→ 坐标转换 world→screen
       │    │    │                   └→ ObjectScreen::update(dt)
       │    │    │                        └→ Object::update(dt)
       │    │    │                             ├ 处理 object_to_add_
       │    │    │                             └ 遍历 Player.children_:
       │    │    │                                  ├ SpriteAnim::update(dt)  帧动画
       │    │    │                                  ├ Collider::update(dt)    同步位置
       │    │    │                                  ├ Stats::update(dt)       回蓝
       │    │    │                                  ├ Timer::update(dt)       闪烁计时
       │    │    │                                  ├ WeaponThunder::update() 冷却
       │    │    │                                  └ MoveControl::update(dt) 读输入→设速度
       │    │    │         └→ updateHealthBar()
       │    │    │    ├ autoEscape() / checkState()
       │    │    │    ├ move(dt)         ← 应用 velocity_ 到 position_
       │    │    │    ├ synCamera()      ← 相机跟随
       │    │    │    └ checkIsDead()
       │    │    │
       │    │    ├─ Spawner::update(dt) ← 这里可能 safeAddChild 新 Enemy
       │    │    ├─ Effect::update(dt)
       │    │    └─ Enemy(们)::update(dt)
       │    │         └→ Actor::update(dt)
       │    │              └→ Object::update()
       │    │                   ├ 处理 enemy 的 object_to_add_ (新 MoveControl)
       │    │                   └ 遍历 enemy.children_ → update
       │    │              ├ aimTarget()  ← 没 MoveControl 就自动追踪
       │    │              ├ move(dt)
       │    │              ├ attack()
       │    │              ├ checkState()
       │    │              └ remove()
       │    │
       │    └─ 3️⃣ children_screen_ 遍历（暂停时也执行）
       │         ├─ HUDStats::update(dt)
       │         ├─ HUDText::update(dt)
       │         ├─ HUDButton::update(dt)
       │         └─ UIMouse::update(dt)
       │
       ├─ updateScore()
       ├─ checkButtonPause/Restart/Back()
       └─ checkEndTimer()
```

### 3. render 阶段

```
Game::run()
  └→ SceneMain::render()
       ├─ renderBackground()  // 绘制网格和边界
       │
       └─ Scene::render()
            ├─ 1️⃣ Object::render() → children_ 遍历（Timer 无渲染）
            │
            ├─ 2️⃣ children_world_ 遍历（世界坐标 → 减去 camera 偏移）
            │    ├─ BgStar::render()
            │    ├─ Player::render()
            │    │    └→ Actor::render()
            │    │         └→ Object::render()
            │    │              └→ 遍历 children_:
            │    │                   ├ SpriteAnim::render() ← 绘制精灵
            │    │                   └ ...
            │    ├─ Enemy::render()
            │    └─ Effect::render()
            │
            └─ 3️⃣ children_screen_ 遍历（屏幕坐标，不加相机偏移）
                 ├─ HUDStats::render()
                 ├─ HUDText::render()
                 ├─ HUDButton::render()
                 └─ UIMouse::render()  ← 最后绘制，保证鼠标在最上层
```

## 五、Spawner 动态挂载 Enemy 的特殊链路

这是最绕的一条链，Enemy **不是直接**加进 Scene 的：

```
Spawner::update()
  │
  ├─ Enemy::addEnemyChild(nullptr, pos, target)
  │    └→ new Enemy(), init(), setTarget()
  │       parent 是 nullptr → 不调用 addChild  ← Enemy 此时是孤儿！
  │
  └─ Effect::addEffectChild(scene, "184_3.png", pos, 1.0f, enemy)
       └→ new Effect(), init()
          effect->setNextObject(enemy)   ← Enemy 暂存在 Effect 里
          scene->addChild(effect)        ← Effect 立即进入 Scene::children_world_
       │
       └→ 下一帧 Effect::update(dt)
            └→ checkFinish()
                 动画播完了 →
                   scene->safeAddChild(enemy)  ← Enemy 终于进入 Scene！
                   （用 safeAddChild 是因为此刻正在遍历 children_world_）
```

这个设计是为了**先播放生成特效，特效结束后敌人才出现**。Effect 充当了"延迟投放器"。

## 六、总结：一条数据流串起来

```
初始化:
  SceneMain::init()
    ├ new Player(), addChild(player) → Scene::children_world_
    │   Player::init() 内部:
    │     safeAddChild(sprite_anim) → Player::children_
    │     safeAddChild(collider)    → Player::children_
    │     safeAddChild(stats)       → Player::children_
    │     setMoveControl(mc)        → safeAddChild(mc) → Player::children_
    │                                  setParent(this) → mc->parent_ = Player
    ├ new Spawner(), addChild(spawner) → Scene::children_world_
    └ addChild(hud)  → Scene::children_screen_

运行时每帧:
  Scene::update(dt)
    → 遍历 children_world_
      → Player::update(dt)
        → Object::update(dt) 遍历 Player::children_
          → MoveControl::update(dt)
            parent_->setVelocity(...)  ← 修改 Player 的速度
        → move(dt)  ← Player 应用速度
      → Spawner::update(dt)
        → Enemy::addEnemyChild(nullptr, ...)  ← 创建孤儿 Enemy
        → Effect::addEffectChild(scene, ..., enemy)  ← Effect 进 Scene，持有 Enemy
      → Effect::update(dt)
        → 动画结束 → scene->safeAddChild(enemy)  ← Enemy 正式进入 Scene
```

MoveControl 就是这样"融入"的：它通过 `safeAddChild` 进入 Player 的 `children_`，引擎递归遍历时自然调用到它；它又通过 `parent_` 指针反向操控 Player 的速度。两条线缺一不可。
